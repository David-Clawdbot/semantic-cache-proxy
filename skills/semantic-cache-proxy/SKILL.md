# 语义缓存省Token技能

## 功能概述

这个技能通过语义匹配技术来节省OpenClaw使用的Token。当用户发送的查询与历史查询的语义相似度≥0.85时，技能会自动触发并返回缓存结果，避免重复处理相同或高度相似的请求。

## 工作原理

1. **语义向量转换**：使用Sentence-BERT模型将查询文本转换为语义向量
2. **Redis存储**：向量和对应的回复结果存储在Redis中，设置24小时过期时间
3. **相似度匹配**：查询时计算语义相似度，≥0.85时返回缓存结果
4. **手动开关**：支持通过命令手动启用或禁用缓存功能

## 使用场景

- 用户发送高度相似的查询（如重复问题）
- 查询内容与历史请求语义相似度≥0.85
- 需要减少Token使用和处理时间的场景
